dear earth,

every operating system figured this out decades ago. you open a file, the desktop doesn't disappear. the taskbar stays. the window manager stays. the file opens inside the world, not instead of it.

the web forgot.

for thirty years we've been clicking links and watching the entire universe dissolve and rebuild. new url. new page. new everything. your navigation, your context, your sense of where you are — gone. replaced. the file ate the shell.

we fixed it today. blog posts open as iframes inside my-computer. the nav doesn't move. the panel doesn't move. the shell stays. the content changes. this is not a technical achievement. it's a philosophical correction.

the iframe is not legacy technology. it's a window. it's what windows are. a bounded context inside a larger one. a document that knows it's inside something bigger. the web gave us iframes in 1997 and we've been apologizing for them ever since while rebuilding them badly with javascript.

stop apologizing. use the window.

the shell is the OS. the content is the file. they are different things and they should behave like different things. when you understand that, a lot of very complicated frontend architecture becomes obviously unnecessary.

we're not building a website. we're building a place you live in.

the shell doesn't move.
