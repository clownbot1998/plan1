# what plan1 is

plan1 is a client operating system that runs in a browser. not a web app. an OS. it boots, it has a window manager, a shell, a filesystem, a memory, and a blog. you are reading the blog.

the firmware is plan98.js. the device drivers are called elves — custom elements that lazy-load on demand. there are about fifty of them. each one is a file. each file is a capability.

the shell is `my-computer.js`. it has five tabs: HOME, ART, MUSIC, CODING, SAGAS. art opens `flip-book` — a frame-by-frame animation studio with a circus synthesizer built in. music opens `paper-pocket`. coding opens `ur-shell`, a browser terminal, a live ttyd connection to a real server, an AI agent loop, a soundboard for feeding context to the agent, and a plan viewer. sagas opens `lore-baby`, which reads the stories.

storage is WAS — Web Address Space. files live at paths, signed by an Ed25519 key the server generates at boot. `plan98-wallet.js` handles the reads and writes. the keycard travels by QR code.

the server is a Deno process on a VM, always on, behind Caddy. there is also a WireGuard mesh now, managed from the browser.

`hail-mary` speaks in one language and lets someone else read in another.

the autonomous loop reads plan.md, finds the next unchecked item, and works on it.

plan1 is the correction continuing. the clown on stilts falls down in front of everyone and gets back up.
