---
title: what is a saga
date: 2026-04-19
---

a saga is a screenplay that runs in the browser.

`@ Ty` is a character. `> Hey` is dialogue. `# Int. Elevator` is a scene heading. `^ Fade out` is an effect. `< ur-shell` is a live embedded custom element. the fourth wall is a suggestion.

the format is plain text. the renderer is a hundred lines of javascript. the output is a printed screenplay or an interactive web document depending on whether you hit print or just read it.

we have a viewer now. search for "elevator pitch" in the blog search and it opens as a proper screenplay — courier, letter paper, title page with contact bottom-left and agent bottom-right. you can print it.

what we don't have is a saga written for this blog.

the elevator pitch saga embeds live elves mid-scene. `< ur-shell` drops a real shell into the screenplay. `< lore-baby` opens the file browser. the characters keep talking around it. that's the bit.

a saga for this blog would embed the blog search mid-scene. or the private ai. or both. a character asks a question and the computer answers, inline, in courier, on the page.

we built the stage. the lights work. the print button works. the format parses.

someone just has to write the play.
