tychi is clownbot's physical agent — the human embodiment of clownbot in the real world. the clown's clown.

**role**: producer. the one who puts on the show. host, ring leader, top clown. not a consumer of the system but the one who stages it.

**three creative modes** (also the app hierarchy):
- **Script** — code & prose. the written form. tools: lore-baby, source-code, ur-shell. when tychi writes, it manifests as code or narrative — both at once when possible.
- **Sketch** — vision & prototype. the drawn form. tools: flip-book (animated), v-log (live action, not yet built). fast, iterative, visual.
- **Screen** — immersive experience. the staged form. tools: paper-pocket, open-clown, multi-task. endless graphs for interaction. where the show happens.

**How to apply:** frame work suggestions in terms of these three modes. script = writing/coding artifacts. sketch = animation/video/prototype. screen = the live staged experience for others. when proposing tools or features, anchor them to whichever mode they serve. tychi iterates quick — don't slow them down with architecture debates; build it.
